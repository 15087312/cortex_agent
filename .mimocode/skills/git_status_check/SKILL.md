# Git 状态检查技能

## 概述
检查项目Git状态，包括未提交更改、最近提交历史和差异统计，帮助了解项目当前版本控制状态。

## 使用场景
- 提交代码前检查状态
- 了解项目最近的修改历史
- 查看有哪些文件被修改
- 检查是否有未跟踪的文件

## 工作流程

### 步骤 1: 检查当前状态
```bash
git status
```
查看：
- 当前分支
- 未暂存的更改
- 已暂存的更改
- 未跟踪的文件

### 步骤 2: 查看最近提交历史
```bash
git log --oneline -10
```
查看最近10条提交记录，了解项目修改历史。

### 步骤 3: 查看差异统计
```bash
git diff --stat
```
查看未暂存文件的修改统计（添加/删除行数）。

### 步骤 4: 查看已暂存差异
```bash
git diff --cached --stat
```
查看已暂存文件的修改统计。

### 步骤 5: 查看未跟踪文件
```bash
git status --short
```
以简洁格式显示所有状态。

## 输出格式

```
- 删除文件: Z 个

### 最近提交
1. abc1234 - 提交信息 (2小时前)
2. def5678 - 提交信息 (1天前)

### 差异统计
- 文件数: N 个
- 添加行数: +XXX
- 删除行数: -YYY
```

## 注意事项
- 检查前确保在正确的目录
- 如果有未提交的更改，建议先提交或暂存
- 定期检查状态避免遗忘修改
