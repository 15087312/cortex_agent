# 记忆索引更新技能

## 概述
更新项目记忆索引（MEMORY.md），记录重要决策、修复和架构变更，帮助团队成员了解项目演进历史。

## 使用场景
- 完成重要功能开发后
- 修复关键bug后
- 做出架构决策后
- 项目里程碑达成后

## 工作流程

### 步骤 1: 读取当前MEMORY.md
```bash
Read /Users/abc/.claude/projects/-Users-abc-PycharmProjects-ai-backend/memory/MEMORY.md
```
了解当前索引结构和已有记录。

### 步骤 2: 确定记录类型
根据工作内容选择记录类型：
- **功能开发**: 新功能实现、优化改进
- **Bug修复**: 问题诊断、修复方案
- **架构决策**: 技术选型、设计变更
- **安全修复**: 漏洞修复、安全改进
- **性能优化**: 性能改进、资源优化

### 步骤 3: 创建记录文件
在 `/Users/abc/.claude/projects/-Users-abc-PycharmProjects-ai-backend/memory/` 目录下创建新的markdown文件。

文件命名规范：
- 功能: `feature_<功能名>.md`
- 修复: `fix_<问题名>.md`
- 决策: `decision_<决策名>.md`
- 优化: `optimization_<优化名>.md`

### 步骤 4: 更新MEMORY.md索引
在MEMORY.md的 `# Memory Index` 部分添加新条目：

```markdown
- [记录标题](文件名.md) — 简短描述（50字以内）
```

### 步骤 5: 验证更新
确认MEMORY.md格式正确，新条目已添加。

## 记录文件模板

```markdown
# [记录标题]

## 背景
简述问题背景或决策背景

## 问题/需求
描述具体问题或需求

## 解决方案
描述采用的解决方案

## 实现细节
关键实现细节（代码片段、配置等）

## 影响范围
- 影响的模块/文件
- 性能影响
- 兼容性考虑

## 验证方法
如何验证解决方案有效

## 相关链接
- 相关文档
- 相关代码
- 相关讨论
```

## 输出格式

```
## 记忆索引更新报告

### 更新内容
- 新增记录: [记录标题](文件名.md)
- 记录类型: 功能开发/Bug修复/架构决策
- 影响范围: 模块/文件列表

### 更新后的MEMORY.md索引
- [已有记录1](file1.md) — 描述1
- [已有记录2](file2.md) — 描述2
- [新增记录](new_file.md) — 新增描述

### 验证结果
✅ MEMORY.md格式正确
✅ 新条目已添加
✅ 文件链接有效
```

## 最佳实践

1. **及时更新**: 完成重要工作后立即更新记忆索引
2. **简洁明了**: 记录标题和描述要简洁，详细内容放在单独文件
3. **分类清晰**: 使用一致的命名规范和分类
4. **关联相关**: 与已有记录建立关联，形成知识网络

## 注意事项
- MEMORY.md是项目的重要知识库，更新时要谨慎
- 确保新记录不与已有记录重复
- 保持索引的整洁和可读性
- 定期回顾和整理记忆索引
