📋 需求

✓ 把感知系统默认关闭
✓ 感知子系统全部默认 True（但主系统关闭时子系统也是关的）
✓ 只是保证打开感知之后全部开启

✅ 实现方案

【1】全局配置层（config/settings.py）

  PERCEPTION_ENABLED: bool = False  ← 默认关闭主系统

  # 子系统配置始终为 True（由主系统控制是否生效）
  PERCEPTION_FILE_ENABLED: bool = True
  PERCEPTION_DIALOG_ENABLED: bool = True
  PERCEPTION_SCREEN_ENABLED: bool = True
  PERCEPTION_VOICE_ENABLED: bool = False
  PERCEPTION_INTERNAL_ENABLED: bool = True

【2】感知管理器层（modules/perception/manager.py）

  class PerceptionManager:
    def __init__(self, enabled: bool = False):  # ← 默认关闭
      # 初始化时从配置读取子系统状态
      self.file_perception = FilePerception(enabled=config.get('file', True))
      self.dialog_perception = DialogPerception(enabled=config.get('dialog', True))
      self.screen_perception = ScreenPerception(enabled=config.get('screen', True))

    def start_monitoring(self):
      # 启动时，重新加载配置中的子系统设置并启用
      self.enabled = True
      apply_subsystem_config()  # 应用配置

    def stop_monitoring(self):
      # 停止时，禁用所有子系统
      self.enabled = False
      self.file_perception.enabled = False
      self.dialog_perception.enabled = False
      self.screen_perception.enabled = False

【3】子系统层（FilePerception/DialogPerception/ScreenPerception）

  每个子系统都支持 enabled 参数，但仅在主系统运行时才生效

📊 行为验证

【场景 1】应用启动（默认）
  ✓ 主系统: False
  ✓ 文件感知: True（启用但不生效）
  ✓ 对话感知: True（启用但不生效）
  ✓ 屏幕感知: True（启用但不生效）
  → 感知系统不运行，性能无影响

【场景 2】手动启动 start_monitoring()
  ✓ 主系统: True
  ✓ 文件感知: True（启用且生效）
  ✓ 对话感知: True（启用且生效）
  ✓ 屏幕感知: True（启用且生效）
  → 所有子系统全部启用，完整感知

【场景 3】手动停止 stop_monitoring()
  ✓ 主系统: False
  ✓ 所有子系统: False
  → 完全停止，无性能开销

【场景 4】通过 API 启动
  POST /api/perception/start
  ✓ 主系统: True
  ✓ 所有已配置的子系统: True
  → 与手动调用行为一致

🔄 向后兼容性

  - 要启用感知，需要：
    a) 修改 settings.PERCEPTION_ENABLED = True，或
    b) 显式调用 start_monitoring()，或
    c) 调用 API 端点启动

📚 使用示例

【1】查看状态】
  from modules.perception import perception_manager
  status = perception_manager.get_status()
  # {
  #   'main_system_enabled': False,
  #   'running': False,
  #   'subsystems': {
  #     'file_perception': True,
  #     'dialog_perception': True,
  #     'screen_perception': True
  #   }
  # }

【2】启动感知系统】
  perception_manager.start_monitoring()
  # 日志: 开始后台监控（启用的子系统: file, dialog, screen）

【3】停止感知系统】
  perception_manager.stop_monitoring()
  # 日志: 停止后台监控（所有子系统已禁用）

【4】禁用特定子系统】
  perception_manager.set_subsystem_enabled("screen", False)
  perception_manager.start_monitoring()
  # 只启用文件和对话感知

【5】通过配置控制】
  # config/settings.py
  PERCEPTION_ENABLED = False  # 保持关闭
  PERCEPTION_SCREEN_ENABLED = False  # 禁用屏幕感知
  PERCEPTION_FILE_ENABLED = True  # 启用文件感知

✨ 关键特性

1. 默认安全性高
   - 应用启动时不运行感知系统
   - 无性能开销
   - 避免自动启动导致的问题

2. 启动灵活快速
   - 一行代码启动：start_monitoring()
   - 或通过配置文件：PERCEPTION_ENABLED = True
   - 或通过 API 端点：POST /api/perception/start

3. 子系统自动启用
   - 启动主系统时，所有已配置的子系统自动启用
   - 无需逐个手动启用
   - 但仍可通过配置选择性启用/禁用

4. 完整的状态管理
   - get_status() 获取完整状态
   - set_subsystem_enabled() 细粒度控制
   - 启动/停止时自动同步子系统状态
