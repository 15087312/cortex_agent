# TUI 快速启动指南

## 🎬 启动 TUI

```bash
# 基础启动（连接 localhost:8080）
python -m cli_tui.main

# 连接远程服务
python -m cli_tui.main --api-url http://remote-server:8080

# 指定模型（可选）
python -m cli_tui.main --model claude-opus-4-6
```

## 📌 核心改进一览

### 改进 1️⃣：持续对话窗口
- ❌ 旧：每次说话都清屏，历史对话丢失
- ✅ 新：对话持续保留，自动滚动，支持查看完整历史

### 改进 2️⃣：自动加载上下文
- ❌ 旧：每次都是冷启动，无法利用历史知识
- ✅ 新：每次自动加载：
  - 最近 5 条对话历史
  - 用户人格特质
  - 当前情绪状态

### 改进 3️⃣：隐去系统提示词
- ❌ 旧：系统消息混杂，影响阅读
- ✅ 新：过滤不必要的系统消息，只保留核心内容

### 改进 4️⃣：记忆检索命令
- ✅ `/search <query>` - 搜索历史对话和知识
- ✅ `/context` - 查看当前加载的上下文

## 🎮 常用命令速查表

| 命令 | 别名 | 功能 |
|------|------|------|
| `/help` | `/h`, `/?` | 查看帮助 |
| `/context` | `/ctx` | 显示当前上下文 |
| `/search <词>` | `/find` | 搜索记忆 |
| `/memory` | `/mem` | 查看记忆状态 |
| `/status` | `/s` | 查看系统状态 |
| `/session <主管>` | `/sess` | 查看会话内容 |
| `/tools` | `/t` | 切换工具面板 |
| `/thinking` | `/th` | 切换思考显示 |
| `/clear` | `/c` | 清空显示 |
| `/export` | `/e` | 导出工具调用 |
| `/exit` | `/q` | 退出 |

## 💡 使用提示

### 常见场景

**场景 1: 查看与某个话题相关的历史**
```
/search Python 异常处理
```
搜索所有与 Python 异常处理相关的过去对话

**场景 2: 了解 AI 当前如何理解你**
```
/context
```
查看 AI 看到的你的人格、情绪和对话历史

**场景 3: 快速检查系统状态**
```
/status
```
查看当前有多少个会话在运行

**场景 4: 查看某个主管的工作**
```
/session 代码主管
```
查看代码主管与哪些专家交互过

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `↑` | 历史回退（在输入框有焦点时） |
| `↓` | 历史前进（在输入框有焦点时） |
| `Ctrl+C` | 退出 TUI |

## 🔧 调试技巧

### 查看完整的 AI 思考过程
```
/thinking
```
默认关闭以保持界面清洁，打开后可看到 AI 的完整思考过程

### 追踪工具调用
```
/tools
```
打开右侧工具面板，查看所有工具调用的详情和耗时

### 导出工具调用追踪
```
/export
```
生成 JSON 文件，可用于分析 AI 的决策过程和性能

## 🐛 故障排查

### Q: 对话没有上下文？
A: 确保后端的记忆 API 正常运行
```bash
curl http://localhost:8080/memory/short-term/context
```

### Q: 搜索没有返回结果？
A: 确保有足够的历史对话
```bash
/memory  # 查看短期记忆和黑盒日志大小
```

### Q: 命令不起作用？
A: 检查命令语法
```bash
/help  # 查看所有命令的正确格式
```

### Q: 性能变慢？
A: 减少显示的消息数量
```bash
/clear  # 清空旧的对话历史
```

## 🎯 最佳实践

2. **使用 /search** - 快速找到相关历史

4. **定期 /export** - 备份重要的工具调用记录
5. **使用命令别名** - `/h` 比 `/help` 更快
