# production_plugin_package

Example production package layout for `modules/plugin_system`.

This directory is a structure example. It does not contain real production evidence.

## Files

- `plugin.yaml` is the plugin manifest.
- `manifest.lock.example` shows where file hashes and dependency locks would be recorded.
- `sbom.cdx.json.example` shows SBOM shape only; it is not a real SBOM scan.
- `scanner_report.example.json` shows scanner evidence shape only; it is not a real vulnerability report and must not be treated as production scanner evidence.
- `SIGNATURE.example` is not a valid signature.
- `src/main.py` is a minimal tool implementation.
- `config_schema.json` is a bounded empty config schema.
- `tests/README.md` is a placeholder for package tests.

Real production packages must be generated by a build/sign/scan workflow. Do not commit private keys. Do not rename example files to production evidence unless the artifact was produced and verified by the real workflow.

python -m modules.plugin_system.manifest_lint . --json

python -m modules.plugin_system.production_policy_check . --dev-mode --json
